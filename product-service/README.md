# 🛍️ Product Service

## 📌 Overview

The **Product Service** is responsible for managing product-related operations such as creating, retrieving, and deleting products.
It is a part of a microservices-based architecture and registers itself with Eureka for service discovery.

---

## 🧱 Tech Stack

* Java 17
* Spring Boot 3
* Spring Data JPA
* H2 Database (Development)
* Spring Cloud Netflix Eureka
* OpenAPI (Swagger)

---

## 📁 Project Structure

```plaintext
src/main/java/com/platform/product_service
│
├── controller        → Handles HTTP requests (API layer)
├── service           → Business logic layer
├── repository        → Database access layer (JPA)
├── entity            → JPA entities (DB models)
├── config            → Configuration classes (Swagger, Security, etc.)
├── dto (optional)    → Request/Response objects (recommended)
├── exception (optional) → Global exception handling
│
└── ProductServiceApplication.java → Main Spring Boot entry point
```

---

## 🧠 Role of Important Classes

### 🔹 Controller Layer

* **ProductController**

    * Exposes REST endpoints (`/products`)
    * Handles HTTP requests & responses
    * Calls service layer
    * Annotated with Swagger annotations

---

### 🔹 Service Layer

* **ProductService**

    * Contains business logic
    * Acts as a bridge between controller and repository
    * Handles validations and rules

---

### 🔹 Repository Layer

* **ProductRepository**

    * Extends `JpaRepository`
    * Handles database operations (CRUD)
    * No implementation required (auto-generated by Spring)

---

### 🔹 Entity Layer

* **Product**

    * Represents database table
    * Annotated with `@Entity`
    * Contains fields like id, name, price

---

### 🔹 Configuration Layer

* **OpenApiConfig**

    * Configures Swagger/OpenAPI documentation
    * Defines API metadata (title, version, description)

---

### 🔹 Main Class

* **ProductServiceApplication**

    * Entry point of Spring Boot application
    * Contains `@SpringBootApplication`
    * Starts embedded server

---

## 🔄 Request Flow

Client → Controller → Service → Repository → Database
Database → Repository → Service → Controller → Client

---

## 🌐 API Endpoints

| Method | Endpoint       | Description       |
| ------ | -------------- | ----------------- |
| GET    | /products      | Get all products  |
| GET    | /products/{id} | Get product by ID |
| POST   | /products      | Create a product  |
| DELETE | /products/{id} | Delete a product  |

## ⚠️ Notes

* H2 database is used for development only
* Replace with MySQL/PostgreSQL in production
* Swagger UI should be publicly accessible (configure later with Spring Security)

## 💡 Best Practices Followed

* Layered architecture (Controller → Service → Repository)
* Constructor-based dependency injection
* Proper HTTP status handling using `ResponseEntity`
* API documentation using Swagger
* Input validation using `@Valid`
